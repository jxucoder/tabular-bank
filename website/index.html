Overview

Generation pipeline

The generator follows a fixed sequence. Each stage is deterministic given the secret and round id, but unknown in advance without the secret.

  1. 1

    Derive the round seed

    The master secret and round id are combined with HMAC-SHA256 to produce a round seed. Independent seeds are then derived for feature generation, DAG construction, row sampling, and split generation. 

    round_seed = HMAC_SHA256(master_secret, round_id)
feature_seed = HMAC(round_seed, "features-0")
dag_seed = HMAC(round_seed, "dag-0")
data_seed = HMAC(round_seed, "dataset-0")
split_seed = HMAC(round_seed, "split-0")
  2. 2

    Sample a scenario

    A scenario is drawn from continuous parameter ranges rather than a short library of fixed templates. This determines problem type, feature count, sample size, missingness, and difficulty.

  3. 3

    Generate feature specifications

    Informative features receive opaque names, types, and either distribution parameters or categorical probabilities. The target column is generated from the chosen problem type.

  4. 4

    Build a causal DAG

    The graph stage samples parent sets, functional forms, coefficients, latent confounders, and optional autocorrelation on root nodes.

  5. 5

    Sample rows and apply missingness

    Rows are generated in topological order. Root signals are sampled first, child nodes inherit transformed parent effects, and the target is sampled last. Missing values are then injected without masking the target column.

  6. 6

    Write benchmark artifacts

    The final dataset is saved together with metadata and repeated train/test folds so benchmark code can consume it directly.
