A contamination-resistant benchmark for tabular machine learning.

tabular-bank replaces fixed public benchmark corpora with secret-seeded benchmark rounds generated by a public engine. Each round yields datasets, repeated splits, round metadata, validation artifacts, official baseline runs, and a publishable leaderboard.

Abstract

Problem statement and core claim

Abstract

Public tabular benchmarks are widely used, but their fixed datasets are vulnerable to memorization, contamination, and benchmark staleness. tabular-bank addresses this by publishing the generation engine rather than a fixed benchmark corpus. A benchmark round is specified by a round identifier and a secret seed, from which tasks, splits, and metadata are deterministically derived. The resulting evaluation workflow supports official manifests, validation reports, checked-in baselines, and a static leaderboard suitable for archival publication.

Primary contributions

  • Secret-seeded round generation instead of a static public dataset suite.
  • Inspectable benchmark artifacts: manifest, validation, splits, and run outputs.
  • Leaderboard publication that separates task metrics from board ordering.

Benchmark framing

  • Generation engine is public and auditable.
  • Round-specific data remains unpredictable without the secret.
  • Leaderboard policy is explicit about tracks and partial coverage.

Motivation

From fixed benchmark corpus to reproducible benchmark protocol

Failure mode of static corpora

A benchmark built from the same public tasks year after year becomes easy to memorize, easy to overfit indirectly, and hard to interpret once the model ecosystem changes.

  • Training contamination can blur the meaning of downstream gains.
  • Leaderboard updates outlive the diagnostic value of the task suite.
  • It becomes difficult to separate generalization from data familiarity.

tabular-bank response

The benchmark becomes a protocol: generate a round, validate the contract, run a fixed baseline set, and publish the resulting board.

  • Rounds are reproducible once the seed is disclosed.
  • Artifacts are inspectable without disclosing future task content.
  • Leaderboard publication is a deterministic build step.

Protocol

Round generation and publication pipeline

The benchmark protocol is intended to be legible. Each stage yields a concrete artifact that can be inspected independently of the final board.

Artifacts

Published round outputs

Core files

Artifact Role
round_manifest.json Authoritative task ordering, protocol summary, and hashes.
validation_report.json Round-level checks, warnings, and manifest fingerprint.
official_baselines/results.csv Split-level scores, fit time, and predict time.
official_baselines/methods.json Track, coverage, and availability metadata per method.

Interpretation

The benchmark is not only a leaderboard. It is a bundle of generation, validation, and evaluation artifacts. That distinction matters if the goal is scientific reuse rather than leaderboard display alone.

  • Manifest makes the round structurally explicit.
  • Validation report captures protocol health, not just model outcome.
  • Run artifacts support re-analysis of ranking behavior.

Evaluation

Task metrics and board ordering

The benchmark is multi-metric at the task level, but single-metric for leaderboard ordering. Binary tasks use ROC-AUC, multiclass tasks use negative log-loss, and regression tasks use negative RMSE; the board is ordered by Elo.

Ranking procedure

  1. Average each model’s scores within each task across repeats and folds.
  2. Construct a model-by-task score matrix.
  3. Compute average rank, win rate, mean score, and Elo.
  4. Sort the board by Elo and expose explicit 1-based rank.

Track and coverage policy

  • Overall board includes only full-coverage methods.
  • Track pages remain visible for partial-coverage methods.
  • Method metadata distinguishes classical and foundation tracks.

Appendix

Reproducibility interface

The website is only a thin wrapper around the actual benchmark interface. The commands and API below map directly to the repository.

CLI

pip install tabular-bank

tabular-bank generate --round round-001 --secret "your-secret"
tabular-bank validate --round round-001
tabular-bank run-baselines --round round-001
tabular-bank build-board --round round-001 --output-dir ./site

Python

from sklearn.ensemble import RandomForestClassifier, RandomForestRegressor
from tabular_bank.runner import run_benchmark
from tabular_bank.leaderboard import generate_leaderboard

result = run_benchmark(
    models={
        "rf-clf": RandomForestClassifier(n_estimators=100),
        "rf-reg": RandomForestRegressor(n_estimators=100),
    },
    round_id="round-001",
    master_secret="your-secret",
)

print(generate_leaderboard(result))
